面談AI評価ツール

採用面談 1 件あたりの「候補者要約 → 条件凍結 → 質問づくり → 面談内容 → 評価」をローカル PC で完結させるツールです。

このマニュアルについて

左側のメニューで章を切り替えます。各章は 独立したページ として表示され、右側の「このページ内」はその章の小節だけを案内します。右上の 🔍 または Ctrl+K で全ページから横断検索できます。

主な機能

  • 面談ごとに専用 URL を作って、関連情報を 1 ヶ所に集約
  • 履歴書要約・面談内容・評価結果を 貼付 で保存（AI に直接投げる必要なし）
  • 役割（NW / Server / Dev / PMO / ITSupport / Special）ごとに別々の評価条件
  • 面談時点の条件を 凍結 → 後でマスタを変えても過去面談に影響しない
  • 過去面談を 匿名サマリとして集計・分析
  • Excel ファイル（マスタ / 面談者一覧）を自動更新
  • 暗号化バックアップを世代管理

動作モード

本ツールは「貼付モード」で動作します。AI 要約・質問生成・評価は Claude などの AI チャット側で実行し、結果をコピー → このツールに貼付という流れです。

起動方法

プロジェクトフォルダの start-app.bat をダブルクリックするだけです。

初回

  1. Node.js 20 以上 をインストール（nodejs.org から、インストーラの「Add to PATH」にチェック）
  2. プロジェクトフォルダを PC の好きな場所に置く
  3. start-app.bat をダブルクリック
  4. 初回だけ npm install（3〜5 分）と next build（1〜2 分）が走る
  5. ブラウザが自動で開き http://127.0.0.1:3939 が表示される

2 回目以降

start-app.bat ダブルクリック → 数秒でブラウザが開きます。

停止

起動時に開いた コマンドプロンプトで Ctrl+C → Y → Enter。

ブラウザを閉じるだけでは停止しません。コマンドプロンプトでの操作が必要です。

画面構成

左端に アイコン付きサイドバー、上部に タイトル + 新規面談ボタン + テーマ切替 + ユーザーアバターの TopBar、その下に各ページの本文が並びます。

画面全体

テーマ切替（ライト / ダーク）

TopBar 右上に月アイコン 🌙 のトグルボタンがあります。クリックで ライト ⇄ ダークを切り替え、選択はブラウザに保存されます。初回訪問時は OS の設定（prefers-color-scheme）を自動追従します。

ダークモードは長時間ダッシュボードを開きっぱなしにする運用（面談中の面談内容貼付など）で目が疲れにくくなります。

基本フロー：全体像

1 件の面談は 新規面談→①→②→③→④→⑤ の順で進めます。

セッション本体（全セクション）

ステータスの自動更新

各ステップで保存すると、対応するステータスに自動で進みます：

  • 新規作成 → 編集中
  • ② 条件凍結 → 質問公開
  • ④ 面談内容保存 → 面談済
  • ⑤ 評価保存 → 評価済

セクション内ナビ（サイドバー）

セッション本体の左側には ①〜⑤ の縦型ナビが常時追従表示されます。

  • クリックでその工程まで即ジャンプ（スムーススクロール）
  • スクロール位置で現在のセクションを自動ハイライト
  • 各項目の末尾に ✓（入力済） / ○（未入力） のマークが付き、進捗が一目でわかる

共通セクションヘッダー

各セクション（①③④⑤）の先頭には 緑の左ボーダー付きヘッダーバーがあります。保存済みの工程は 保存済 chip が並び、未着手の工程はグレーのままになります。

② 条件凍結だけは専用のヘッダーで、条件チップ群と「修正」ボタンが並びます。

新規面談を作る 編集中

面談ごとに専用の URL を作るのが最初のステップです。

作成手順

  1. 画面右上の 「+ 新規面談」 をクリック
  2. 候補者の 氏名 と、応募 役割（NW / Server / Dev / PMO / ITSupport / Special）を入力
  3. 「作成して開く →」 をクリック
  4. 自動的にセッションページ /sessions/<ID> に遷移

新規作成画面

セッション ID について

ID は YYYYMMDD_HHMMSS_<氏名>_<役割> 形式で自動生成。秒まで含むので同一分内の連続作成でも ID 衝突しません。

役割を間違えても 複製機能 で後から変更できます。

① 面談者情報 編集中

履歴書を AI で要約して、結果をこのエリアに貼ります。API モードでは最大 2 ファイルまで直接アップロードして自動要約できます。

貼付モードの手順（既定）

  1. ① エリア上の 「📋 コピー」 ボタンで履歴書要約プロンプトをクリップボードへ
  2. Claude などの AI チャットに貼付 → 履歴書ファイル（PDF / Word / Excel）を添付
  3. AI が出した要約をすべてコピー（Ctrl+A → Ctrl+C）
  4. ① エリアの「要約」テキスト枠に貼付（Ctrl+V）
  5. フォーカスを外すと 自動保存（右下に「保存済」インジケータ表示）

API モード: 最大 2 ファイルまで自動要約

API モードに切り替えると、履歴書ファイルを直接アップロードして AI に要約してもらえます。履歴書 + 職務経歴書のように 2 種類のファイルを合わせて 1 名分の要約に統合するのが典型的な使い方です。

  • 対応形式: PDF / Word (.doc / .docx) / Excel (.xlsx / .xls)
  • 1 ファイル上限: 3.7 MB（Server Action の 5 MB body 制限に収まるよう base64 換算で 3.7 MB）
  • ファイル数上限: 2 ファイルまで（既定「履歴書」+「職務経歴書」を想定）
  • UI: 1 個目を選ぶと積み上げリストに表示、追加ボタンが「📎 履歴書をアップ（2/2）」に変わる。2 個目を入れると追加ボタンが消える
  • 取消: 各ファイル行の 「取消」 ボタンで個別に外せる
  • 要約時: 複数ファイル時は AI プロンプトに #添付 1 / #添付 2 の見出しが付き、AI が区別して統合要約する

旧 .xls ファイルはブラウザ内で自動的に .xlsx に変換されてから送信されます（サーバー側では扱えないため）。変換中は「.xls をブラウザで .xlsx に変換中…」表示。

Excel 自動分割

テキスト内に「経歴サマリ：」「主要スキル：」「強み：」の見出しを入れると、Excel エクスポート時に自動で 3 列に分割されます。

② 条件を凍結する → 質問公開

求人情報の条件と評価軸をこの面談に固定します。

何をするのか

求人情報の「求める人材条件」と「評価条件」を、この面談に スナップショットとして固定します。後でマスタを変えてもこの面談には影響しません。

手順

  1. ② エリアで「現在の条件」を確認
  2. 下部の 「この内容で凍結する」 ボタンをクリック
  3. 確認ダイアログで「OK」
  4. ステータスが 質問公開 に変わり、③ 質問エリアが触れるようになる

「凍結」＝今の評価条件を写真のように コピーして保存すること。

あとでマスタの重み・軸・合格ラインを変えても、この面談は当時の条件で評価され続けます。

例：マスタで「NW はコミュニケーション力の重みを 3 → 2 に下げる」設定にしていれば、その面談のコミュニケーション力の重みは 2 で固定。
あとから NW の重みを 3 に戻しても、この面談だけは 4 で計算されます。

凍結後に条件を修正したい場合

凍結済ヘッダー右側の 「修正」 ボタンで編集モードに入れます。人材条件・評価軸・重み・合格ラインを直接書き換え、下部の 「この内容で再凍結する」 ボタンで凍結条件が上書きされます（マスタは変わりません）。誤入力・軸抜けを直したいときに利用してください。

⑤ 評価保存後は凍結条件を変えると 過去スコアの意味が変わる ため、原則として評価前に完結させます。

③ 質問を貼る 質問公開

候補者の経歴と条件に基づく面談質問を AI で生成、結果を貼ります。

手順

  1. ③ エリア上の 「📋 コピー」 ボタンで質問生成プロンプトをクリップボードへ
  2. Claude などの AI に貼付 → 候補者の経歴と組み合わせて 15問前後を生成
  3. AI 回答をコピーして ③ エリアの 「人間性」 枠と 「技術」 枠に振り分けて貼付
    • AI が ## 人間性 / ## 技術 の見出しを付けている場合は 1 枠に全文貼付でも自動振り分けされます
  4. 「保存」 をクリック → 下の「構造化プレビュー」で 1 問ずつカード表示される

質問の形式

質問テキスト内で以下の接頭辞・キーワードが使えます（AI に生成させる時点でこの形式にしておくとカードに反映されます）:

  • ⭐（行頭）：必須質問マーク。カードに ⭐ が表示される
  • Q1. / T1. のような Q / T + 番号：質問見出し
  • 狙い: 〜：質問の意図
  • 解答例: 〜：参考解答

編集はテキスト枠内で直接行います（並べ替え・削除も文字ベース）。保存後のカードは表示専用です。

質問数の目安

既定は 人間性 7 問 + 技術 8 問。カウンタが目標値のとき緑、超過は黄で色付けされます。プロンプトを AI に流す前に文中の数字を書き換えれば任意の件数で生成できます。

コピー系ボタン

  • 本文コピー：狙い / 解答例を除いた質問本文のみをコピー（面談本番の司会用）
  • 全文コピー：狙い / 解答例を含めて全部コピー（面談官への配布用）

④ 面談内容を貼る → 面談済

面談本番の文字起こしや手書きメモをここに貼り付けます。

手順

  1. 面談本番は Teams など別ツールで実施 → 文字起こし
  2. 文字起こし（または手書きメモ）を全選択コピー
  3. ④ エリアに貼付 → 「保存」
  4. ステータスが 面談済 に変わる

面談内容の例（貼付イメージ）

フォーマットは自由（Teams 文字起こしそのまま／手書きメモ／整形済みどれでも可）。上限は約 200 万文字なので、通常の 60 分面談なら余裕で収まります。参考までに整形版の例：

日時: 2026-06-30(月) 14:00-15:00
面談官: 採用担当 A / 現場リード B
候補者: 測試 太郎（Test 役割応募）
形式: Teams（録画あり）

⑤ 評価を貼る → 評価済

面談内容と凍結条件から AI に評価 JSON を生成してもらい、結果を貼ります。

手順

  1. ⑤ エリアの「評価結果 JSON を貼り付ける」を開き、「📋 コピー」 ボタンで評価プロンプトをクリップボードへ
  2. Claude などの AI に貼付 → 面談内容 + 凍結条件を組み合わせて JSON 形式で評価結果を生成
  3. AI が返す JSON を評価 JSON 枠に貼付 → 「保存」
    • AI が ```json のフェンスで囲んでいたら囲みを除いてから貼ってください（素の JSON のみ受け付けます）
  4. 各軸スコア・総合スコア・合否（合格 / 普通 / 不合格）が自動表示
  5. 最後に 採否（採用 / 不採用 / 未確定）を選ぶ → ステータスが 評価済 に

注意

凍結条件を変えるとスコアの意味が変わるため、評価後の凍結条件変更は基本不可です。

ステータスの流れ

セッションは 4 つのステータスを順に進みます。

一覧

ステータス意味次に進む条件
編集中新規作成直後、まだ条件凍結前②で条件凍結
質問公開条件凍結済み、質問が編集可能④面談内容を保存
面談済面談内容が保存された⑤評価を保存
評価済評価が確定し、採否が決まった—（終了）

採否

  • 採用 — 内定
  • 不採用 — 見送り
  • 未確定 — 判断保留中

一覧画面

/list はすべてのセッションを一覧します。

画面

フィルタ

  • 状態：編集中 / 質問公開 / 面談済 / 評価済
  • 役割：NW / Server / Dev / PMO / ITSupport / Special（マスタで追加した役割も表示）
  • 合否：採用 / 不採用 / 未確定
  • 氏名検索：上部の検索ボックスに入力（部分一致）

各行の操作

  • 詳細 → セッション本体に遷移
  • 削除 → ゴミ箱に移動（即時完全削除ではない）
  • チェックボックス（評価済のみ）→ 複数選択して横断比較へ

複数選択時の操作（画面下部の固定バー）

  • N 件を一括削除 — 選択した全セッションをゴミ箱へ
  • N 件を比較 → — 横断比較画面 へ（2 件以上で有効）
  • 選択解除 — チェックを全解除

複製と役割変更

同じ候補者を別の役割で評価し直したい、役割を間違えたときに使います。

手順

  1. 複製元のセッション詳細画面を開く（一覧の「詳細」から）
  2. ヘッダー右側の 複製アイコン（コピーマーク）をクリック
  3. ダイアログで氏名・役割を変更可能
  4. 「複製する」 をクリック → 新セッションが自動で開かれる

引き継ぎルール

① 候補者要約✓ 引き継ぎ✓ 引き継ぎ
uploads/ 履歴書ファイル✓ 引き継ぎ✓ 引き継ぎ
② 凍結条件✓ 引き継ぎ✗ 破棄（新役割向けに再凍結）

💴 コスト試算

/cost で LLM API 呼び出しの概算コストを集計します。

表示されるタイミング

API モードで要約 / 質問生成 / 面談内容 / 評価を実行すると、その度に呼び出し記録が蓄積されます。貼付モードのみで運用している間は /cost は「呼び出し記録がまだありません」と表示されます。

画面

画面の見方

  • KPI カード：累計金額 / 呼び出し回数 / 累計トークン / 1 回あたり平均
  • 月別：呼び出し月ごとの合計
  • プロバイダ別：Anthropic / OpenAI / Google の合計
  • モデル別：モデル ID ごとの合計
  • 工程別：① 要約 / ③ 質問生成 / ③ 質問整形 / ④ 面談内容要約 / ⑤ 評価 の合計
  • 直近呼び出し：最新 50 件の明細

すべての集計表の右端に「1 回あたり」列があり、その工程・モデル・プロバイダの平均単価が一目でわかります。

概算のロジック

  • 文字数 → token 換算：1 token ≈ 2.5 文字（日本語想定）
  • 為替：USD $1 = ¥160 固定
  • 単価表：2026-06 時点の公開単価（lib/pricing.ts）
  • キャッシュ割引・Batch 割引は未計算のため「上限見積り」として扱ってください

1 面談あたりの目安（既定モデル Claude Sonnet 4.6）

工程入力目安出力目安1 回の JPY
① 要約3〜15 K字0.8〜1.5 K字¥1〜5
③ 質問生成（15問）2.5〜4.5 K字10〜17 K字¥10〜18
③ 質問整形（任意）2〜4 K字2〜5 K字¥2〜6
④ 面談内容要約（任意）5〜20 K字≤1500 字¥2〜5
⑤ 評価5〜15 K字5〜9 K字¥6〜11

必須 3 工程（① 要約 + ③ 生成 + ⑤ 評価）のみで ¥17〜33 / 面談、整形・面談内容要約を含めたフル構成でも ¥22〜45 / 面談が目安です。月 100 面談で ¥2,500〜4,500 程度。

モデルを Haiku 4.5（$1/$5）に落とすと約 1/3、Opus 4.7（$15/$75）に上げると約 5 倍になります。

記録される情報

各 API 呼び出しの provider / model / 入力文字数 / 出力文字数 が data/logs/audit.log に記録されます（PII なし）。/cost はこのログを走査して集計するため、audit.log を消すと集計もリセットされます。

求人情報管理

/master で 求人情報 と 評価条件 を編集します。

画面

求人情報

  • 役割の 追加 / 編集 / 削除
  • 各役割の「求める人材条件」（基本人物像 + 未経験者必須）
  • 経験要件・未経験可フラグ

役割 ID は NW Server Dev 等の英数字（半角英数 + ハイフン / アンダーバーのみ）。

評価条件（BARS）— 5 軸バランス型

既定は以下の 5 軸。総合スコアはこれらの重み付き平均で自動計算されます。

軸意味既定重み質問例
技術力職種スキル・専門知識・実装力3〜4「得意な言語で気をつけていること」
問題解決力分析・課題設定・打ち手選定3〜4「困難だった場面と乗り越え方」
主体性 / 意欲自走・提案・オーナーシップ3「自分から提案して改善した事例」
コミュニケーション力説明力・傾聴・調整3「対人でこじれた場面の解決」
学習意欲 / 成長性キャッチアップ速度、学び方3「新技術を独学で身につけた経験」
  • 重みレンジ：1〜5（既定 3〜4）
  • 合格ライン / 普通ライン：総合スコアのしきい値（既定 合格 4.2 / 普通 3.5）
  • スケール：0.3〜5.0、0.5 刻み、全 10 段階
  • 軸の追加・削除・改名は /master の評価条件エディタから可能

役割別の重み上書き（既定セット）

役割ごとに重み・合格ラインを差し替えできます。既定は以下の通り。

役割技術力問題解決主体性コミュ学習合格ライン
NW443234.2
Server443234.0
Dev543334.2
PMO234433.8
ITSupport333433.6
Special333334.3

役割別に上書きされた重みは ④凍結時にセッションに スナップショットとして固定されます。あとからマスタの重みを変えても過去の面談スコアの意味は変わりません。

編集不可 フラグ

役割単位で 「編集不可」チェックボックスを ON にすると、その役割で作成された新規面談は以下のように動作します。

  • 新規セッション作成時に ④ 求める人材条件が自動で凍結される（conditions_snapshot.json を即時生成）
  • セッション画面の 「修正」ボタンが無効化され、条件を書き換えできない
  • ホバーで「マスタで編集不可に設定されているため修正できません」と理由が表示される

用途: 契約先が決めた評価条件を面談官が勝手に変えないよう固定したい場合、監査観点で凍結条件の一貫性を保証したい場合など。

編集不可の役割は 🔒 アイコンが表示されます（一覧表・新規面談の役割選択・セッションの ② ヘッダ）。

エクスポート

マスタ全体（役割 + 評価条件）を JSON / Excel ファイルとして書き出しできます。編集前のスナップショット保存や、他 PC への引き継ぎ元として利用してください。

  • 求人情報セクション右上の 「エクスポート」 ボタン — 求人情報のみ JSON
  • ページ末尾の 「マスタ一括 export」：「📥 JSON」 — 役割 + 評価条件をまとめて JSON
  • 同じく 「📊 Excel」 — テンプレ準拠の .xlsx（人事担当への配布用）

読み込み（インポート）は現状 UI 未提供です。API 経由（POST /api/master/import）または JSON ファイル直置きで反映してください。

設定

/settings で全体設定を管理します。

画面

データ保存先（dataRoot）

面談データ・マスタ・ログの保存先。既定は ./data。 システムディレクトリ（C:\Windows 等）は拒否されます。

保存期間（自動削除）

判定確定日（closedAt）からの経過で自動削除する日数を設定します。0 = 自動削除しない。

  • 採用：既定 0
  • 不採用：既定 180 日
  • 未確定：既定 0（通常 0 推奨）
  • ゴミ箱の猶予日数：ソフト削除後にこの日数で完全削除（既定 7 日）
  • 匿名サマリを analytics/ に残す：完全削除時も統計データだけ残すオプション

スイッチ「自動削除を有効化」が OFF の間は日数を入れても実行されません。手動実行は下の「保存期間スイープ」から。

バックアップ世代の保持

  • バックアップ保存日数：既定 90 日（0 = 自動削除しない）
  • バックアップ世代上限：既定 0 = 無制限

両条件のいずれかに該当した世代が「世代を整理」実行時に削除されます。

保存期間スイープ

設定した保存期間ルールに従って古いセッションを削除します。実行前に 「次に消える面談を確認」 でプレビュー表示 → 問題なければ「スイープ実行」。
削除されたセッションは ゴミ箱 経由で猶予期間内なら復元可能です。

監査ログビューア

ページ末尾に直近 50 件の操作ログが表示されます（詳細は 監査ログ）。

ゴミ箱と復元

/trash で削除済みセッションを管理します。

画面

2 段階削除

  1. ソフト削除：一覧から「削除」 → data/_trash/ に移動、一覧から消える
  2. 復元：猶予期間内ならゴミ箱から「復元」で戻せる
  3. 完全削除：猶予期間（既定 7 日）超過、または手動「完全削除」で物理削除

注意

完全削除すると 元に戻せません。実行前に必ず氏名を確認してください。

匿名サマリは残せる

設定で「匿名サマリを analytics/ に残す」を ON にすると、完全削除しても 氏名・履歴書・面談内容を含まない統計データは残り、分析画面 で使えます。

バックアップ

/settings 下部の「バックアップ」セクションで管理します。

作成

  1. お好きなパスワードを入力（バックアップの中身を守るため）
  2. 「今すぐバックアップ」 ボタンをクリック
  3. これで完了。バックアップファイルが 1 つ作られます

面談データとマスタの中身をまるごと 1 ファイルにまとめて、入力したパスワードで暗号化して保存します。暗号化は AES-256-GCM、パスワードからの鍵導出は PBKDF2-SHA256（200,000 回反復）。パスワード無しの平文バックアップは作成できません。

パスワードの管理

パスワードを紛失すると 復元不可 です。パスワード管理ツールに保存推奨。

世代管理

  • 保存日数：既定 90 日（0 = 自動削除しない）
  • 世代上限：既定 0 = 無制限
  • 「世代を整理」ボタンで手動掃除可能

復元（手動）

現状、GUI からの復元機能は未提供です（Phase 2 で追加予定）。緊急時は付属の CLI スクリプトで復号 → 手動配置してください。

  1. アプリを停止（コマンドプロンプトで Ctrl+C）
  2. 現在の data\master\ と data\sessions\ を別フォルダにコピーして退避
  3. 復号スクリプトを実行:
    node scripts/decrypt-backup.mjs data\_backups\backup-YYYYMMDD-HHmm.enc.tar.gz restored\
  4. プロンプトに従ってパスワードを入力（Enter で確定）
  5. restored\master\ と restored\sessions\ を data\ 直下へ配置
  6. start-app.bat で再起動

スクリプトは復号だけを行い、既存データは書き換えません。事故防止のため出力先に既に master/ か sessions/ があると上書きせず終了します。

分析画面

/analytics で過去面談を匿名集計します。

画面

表示内容

  • KPI カード：累計件数、合格率、平均総合スコア、役割数
  • 月別の合否分布：closedAt 基準で月ごとに合格 / 普通 / 不合格を積み上げ
  • 役割別の合格率
  • 評価軸ごとの平均スコア（バーグラフ）
  • 明細：匿名 1 件ずつ（最新順、軸スコア横並び）

プライバシー

分析データは 氏名・履歴書・面談内容を含まない匿名サマリです。元の面談を完全削除しても統計だけは長期保存できます。

横断比較

/compare で複数の評価済セッションを並べて比較します。

画面

選んだ人数によって表示レイアウトが 2 種類あります。

2〜6 件：標準ビュー（候補者を列で並べる）

7 件以上：転置ビュー（候補者を行で並べる）

用途

  • 同じ役割で複数候補者を比較 → どの候補者を採用するか
  • 複数役割を横断比較 → 部署横断採用の意思決定
  • 大人数の候補者から絞り込みたいときは 7 件以上選択して転置ビューでソート活用

Excel エクスポート

マスタ・候補者一覧を Excel ファイルとして自動的に生成します。

自動更新ファイル

  • data/exports/マスタ.xlsx — 求人情報 + 評価条件
  • data/exports/面談者一覧.xlsx — 全候補者 1 件 1 行
  • data/sessions/<セッションID>/<セッションID>.xlsx — 面談 1 件ごとの詳細シート（旧 session.xlsx から改名。ファイル名がフォルダ名と一致するので、Explorer でコピペしても衝突しません）

マスタやセッションを保存するたびに 自動再生成 されるので、いつ開いても最新です。旧 session.xlsx が残っているセッションは、次に保存したタイミングで新形式に置き換わり、旧ファイルは自動削除されます。

3 列自動分割

Excel 出力時、① 候補者要約は「経歴サマリ：」「主要スキル：」「強み：」の見出しで3 列に自動分割されます。

FAQ

よく寄せられる質問。

質問一覧

役割を間違えて作成してしまった
一覧画面で対象セッションの 「複製」 をクリックし、複製ダイアログで役割を変更してください。① 候補者情報は引き継がれ、②③ は新役割向けに自動でクリアされます。
③ 質問エリアに何も入力できない
② 条件凍結 を先に行う必要があります。② で「条件を凍結する」をクリックしてから ③ に進んでください。
削除した面談を元に戻したい
ヘッダの 「ゴミ箱」（/trash）から復元できます。猶予期間（既定 7 日）を過ぎると完全削除されます。
バックアップはどう取る？
/settings 下部の「バックアップ」で暗号化パスワードを入れて作成。AES-256-GCM（鍵導出は PBKDF2-SHA256 200k）で暗号化されたアーカイブが data/_backups/ に保存されます。パスワードを忘れると復号不可。
複数 PC で同じデータを使いたい
設定で dataRoot を共有フォルダ（OneDrive 等）に指定すれば共有可能。ただし 同時編集はサポートしません（最後に保存した方が勝つ）。
過去の合格率や軸別スコアを見たい
分析画面 で集計表示されます。匿名化されているので個人特定はできません。
データは PC のどこに保存されている？
既定では data/ フォルダ配下（求人情報・面談セッション・監査ログ・バックアップ）。保存先を変えたい場合は /settings の 「データ保存先」 で任意フォルダを指定できます。OneDrive など同期フォルダに変えれば別 PC からも同じデータを開けます。
履歴書が PDF や Word だがそのまま貼れない
貼付モードでは テキスト しか受け付けません。運用としては、Claude や ChatGPT に PDF/Word を投げて 「経歴サマリ / 主要スキル / 強み」 の 3 見出しで要約させ、その結果を ① に貼り付けます。① 候補者情報を貼る にサンプル形式があります。
一度保存した評価やスコアを後から修正できる？
はい、⑤ 評価セクションを開いて上書き保存すれば evaluation.json と一覧の総合スコア・採否 が書き換わります。採否（採用 / 不採用 / 未確定）の変更も同じ手順です。
ただし 分析画面 の集計は保存期間スイープが匿名サマリを書き出したタイミングのスナップショットなので、日常の上書き保存では更新されません（スイープ後は ⑤ の再編集自体ができなくなります）。
面談者が増えて一覧から探しにくい
一覧画面上部のフィルタで 役割 / 状態 / 採否 / 氏名（部分一致） による絞込みが可能。表ヘッダをクリックすれば作成日やスコア順の並び替えもできます。

トラブル時

よくある問題と対処。

ブラウザが自動で開かない

手動で http://127.0.0.1:3939 を開いてください。それでも繋がらない場合：

  • コマンドプロンプトに「Ready」と表示されたか確認
  • ファイアウォール / セキュリティソフトが node.exe をブロックしていないか
  • ポート 3939 が他で使われていないか（netstat -ano | findstr :3939）

Error: spawn EINVAL が出て起動しない

Node.js 22 系以降で next-with-port.mjs が .cmd を spawn するときに発生します（CVE-2024-27980 対策の仕様変更）。
scripts/next-with-port.mjs の spawn オプションに shell: isWin が含まれていることを確認してください。含まれていない場合は追記して保存 → 再度 start-app.bat。

「保存」を押しても反映されない

  • 画面右下に エラーバナーが出ていないか確認
  • F5 でページをリロードして「最終保存：…」のタイムスタンプが更新されたか確認
  • それでもダメなら 監査ログ で該当操作の有無を確認
  • 監査ログにも無ければ、コマンドプロンプトのエラー表示を確認

アプリを完全に止めたい

起動時に開いた コマンドプロンプトで Ctrl+C → Y → Enter。ブラウザを閉じるだけでは停止しません。

データが消えた

  1. まず ゴミ箱 を確認（猶予期間内なら復元可）
  2. data/_backups/ にバックアップがあれば CLI で復号 → 手動配置
  3. OneDrive 等で同期しているなら、サービス側の「履歴」から復元
