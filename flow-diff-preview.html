2. 差分の詳細（3 件）

差分 #1 ・ 意図的な設計差分（設計書 v1.4 §7）

📁 保存形式：Excel ではなく JSON

📋 フローチャート（案）

ローカル Excel

左下に ローカル Excel のシリンダ表示。Excel ファイルで集中管理する想定。

  • 使い慣れた Excel で開ける
  • 1 ファイルで全件管理
✅ 実装

ローカル JSON（面談ごとにフォルダ）

data/sessions/<日時>_<氏名>_<役割>/ の中に各セクションの JSON ファイル。

  • session.json / candidate.json / conditions_snapshot.json / questions.json / minutes.json / evaluation.json
  • セクションごとに独立 → 部分更新が安全
  • マスタとの再現性（凍結スナップショット）を担保

選択理由：Excel は同時編集ロック・部分更新の困難・スキーマ崩れ・PII 混入が起きやすい。JSON はプログラム可読 + Git 履歴で追跡可 + 凍結スナップショット表現に最適。
もし Excel 出力が必要なら：マスタ画面の「📥 エクスポート」と同じ仕組みで、面談一覧 → Excel 書き出しを追加実装可能（半日）。
