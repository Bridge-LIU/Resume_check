変更 §6 評価マスタに「重み」「ロール別オーバーライド」追加

v1.0

"評価軸": ["主体性", "問題解決力", ...]
"合格ライン": 4.2
"普通ライン": 3.5

軸はフラット配列。重みなし。合格/普通ラインは全役割共通。

v2.0

"評価軸": [
  {"名前":"主体性","重み":3}, ...
]
"ロール別": {
  "NW":     {"重み":[3,4,3,3], "合格":4.2, ...},
  "Server": {"重み":[3,3,3,3], "合格":4.0, ...},
  "PMO":    {"重み":[3,3,4,4], "合格":3.8, ...}
}

軸は全役割共通、重み・合格/普通ラインは役割別に上書き可。凍結時は resolveEvalForRole でフラット展開。

変更 §7 ファイル設計 — CP932 化け既知問題を追記

v2.0 追記

  • callable Server Action 経由の非 ASCII 文字列 → CP932 化け（Next.js 16 + Turbopack + Windows）。
  • 例：複製ボタンから渡した日本語名が 闡幄･ｿ閾ｨ豬ｷ蜈ｬ蝨胆ITSupport のように化ける。
  • /new（form action）経由は影響なし。dev サーバ落下を防ぐため lib/crashGuard.ts で全例外を捕捉。

新規 §8.5 コスト試算ダッシュボード /cost (v1.8〜)

v2.0 で新設

  • データソース：data/logs/audit.log から 5 イベント（②候補者要約 / ⑤質問生成・整形 / ⑥議事録 / ⑧評価API）。
  • 換算：tokens ≈ chars / 2.5、¥ = $ × 160 固定。キャッシュ／Batch 割引は無視（上限見積り）。
  • 集計：KPI（累計¥／呼出回数／累計トークン／月平均）＋ 月別／プロバイダ別／モデル別／工程別／直近50件。
  • Phase B：adapter を { text, usage } 返却に refactor して実トークン記録予定。

新規 §9.5 Excel ミラー（JSON → Excel 片方向自動同期）(v1.7〜)

v2.0 で新設

  • 正本は JSON。Excel は読取専用ミラー（書き戻しなし）。テンプレ基準は 大体功能.xlsx。
  • fire-and-forget：JSON 保存成功後に非同期で Excel 更新。失敗は warn のみ。
  • 採用ライブラリ：ExcelJS（SheetJS Community はセルスタイル保持不可のため除外）。
  • 出力：マスタ.xlsx / 候補者一覧.xlsx。手動 DL は ASCII fallback。
  • 用語統一：「応募者」→「候補者」。
  • 既知制約：duplicateSession は直コピーのため複製直後はミラー追従しない。

変更 §11 リスクと対策 — バックアップ暗号化＆マルチプロバイダ PII

v1.0

  • データ消失 → Phase 4 でバックアップ必須化（暗号化方式は未指定）。
  • PII送信 → 既定を貼付に。

v2.0

  • データ消失 → AES-256-GCM + PBKDF2-SHA256 (200K iter)、password 必須（空 password は 400 返却）。
  • PII送信 → マルチプロバイダ対応により「選んだ AI 会社の API に送られる」を明示。案件ごとに「使う1社」を統一する運用を推奨。
