サイドバーナビ — レイアウト案

セッション詳細ページ /sessions/[id] 左の §ジャンプメニュー。現行版+4案を並べた比較。

左に sidebar、右に「ページ中身ダミー」を並べた状態のプレビュー。Active = ⑤質問リスト、状態は ②④⑤⑥=完了、⑧=未着。

現行（今のコード）

フラット。Active 行だけ枠あり、他は何もない状態。

— ページ中身ダミー —

A. Vertical Stepper（縦ステッパー）

番号サークル + 縦の接続線で「面談の流れ」を視覚化。完了は塗り、未着は枠だけ。Active は青リング。

  1. ✓
    ② 面談者情報
    完了
  2. ✓
    ④ 求める人材条件
    完了
  3. ⑤
    ⑤ 質問リスト
    現在地
  4. ✓
    ⑥ 議事録
    完了
  5. ⑧
    ⑧ 評価結果
    未着
— ページ中身ダミー —

⚙ 向き：ワークフローの順序が大事なツールに最適。/sessions/[id] は面談の進行と一致するのでハマる。
⚠ 縦に少しスペース取る（既存より +30px くらい）。

B. アクセントバー（Vercel docs 風）

Active 行の左端に 3px 青バー。フラットだが「いま見ている所」が明確。状態は右の小ドット。

— ページ中身ダミー —

⚙ 向き：いちばん控えめ。情報密度を保ちながら active がはっきり。現行の延長で違和感最小。
✓ 既存コードからの変更も最少。

C. カード型 + 状態バッジ（Notion 風）

各項目を独立カード扱い。状態を「完了」「未着」のテキストバッジで明示。情報密度高い。

— ページ中身ダミー —

⚙ 向き：状態の言葉を残したい場合。一覧表に近いノリでカチッとしてる。
⚠ 縦スペースは A より +20px。56px 幅推奨。

D. 進捗トラッカー（ダッシュボード風）

上部に「全体進捗 4/5」を出してから項目リスト。番号バッジ + アイコン化された状態。

進捗 4 / 5
— ページ中身ダミー —

⚙ 向き：「あとひと工程」みたいなモチベ要素を入れたいとき。番号がアイコン化されて読みやすい。
⚠ A・C より色とアイコンが強い。落ち着いた業務 UI から少し外れる。

推奨

  • B（アクセントバー）：いちばん控えめで、現行コードからの差分が小さい。業務 UI を崩さない。
  • A（Stepper）：面談ツールという文脈にいちばん意味的に合う。「②→④→⑤→⑥→⑧ の流れ」を視覚化できる。
  • D（進捗トラッカー）：ホーム画面のセッション一覧で「進捗 N/5」を出している場合、視覚言語が揃う。
  • C は「悪くないが Notion すぎる」感じ。表中心の本ツールには情報密度が過剰かも。
