インポート時の解決ルール
  1. __DEFAULT__ 行が共通既定を上書き。無ければ既存値を維持
  2. 役割行のセルが空 → 共通既定にフォールバック（重み = 既定の 3、ライン = __DEFAULT__ 行の値）
  3. 役割行のセルに値あり → そのまま採用
  4. 役割行に「ロール」シートに存在しない ID があれば、その行はスキップして警告
  5. 同じロールID が複数行ある → エラーで止める（部分反映なし）
例：上の NewRole 行は全て空欄 → 重み=3, 合格=4.2, 普通=3.5 で取り込まれる（= 共通既定が丸ごと効く）
エクスポート時：現在の JSON 状態をそのままシートに展開。役割が ロール別 に未登録ならその役割行は全空欄で出力（= 共通既定使用と同義）。
JSON 入出力（既存）も並行して残す。Excel はビジネス担当者向け、JSON は開発・バックアップ向け。
